# 処理パイプライン解説（PIPELINE.md）

2026-07-05 時点。「深度を取得して world position を作る」以降に何が行われているかを、
段階ごとに**何を入力し、何を変更し、何を変更しないか**で整理する。

画像を読み込む/Recompute するたびに、下の 1→8 が毎回この順で実行される
（推論 0 だけは画像・モデル・num_tokens が変わったときのみ）。

---

## 0. 推論（js/inference.js）

- 入力: 元画像
- MoGe-2 が **point map（各画素の3D位置の素）/ 法線 / mask（空・不確実領域=0）/
  metric scale** を出力する。解像度はモデル解像度（例 686×518）
- 以後の全処理はこの解像度のグリッドが土台。**テクスチャだけは元画像解像度のまま**

## 1. 後処理 = depth と world position の元を作る（js/moge_post.js `process`）

- point map からカメラの focal / shift を復元し、各画素の **metric depth** と
  **カメラ空間の3D点（points）** を作る
- ここで「1画素 = 1つの深度 = 1つの3D点」が確定する。3D点は必ず
  「そのピクセルのレイ（視線）上」にある。以後の段階で深度を変える場合も、
  点は同じレイ上を前後に動くだけ（画面上の位置はズレない）

## 2. mask の適用（js/moge_post.js `cleanDepthMask`）

- **エッジの削除はもうしていない**（rtol=1 固定で呼ぶ。削除は EdgeSnap に置き換え済み）
- やることは「無効深度（z≤0）の画素を無効にする」+ UI の Sky / Masked Area に応じて
  「mask=0 の画素（空など）を無効にする」だけ
- 出力: 有効/無効の二値マスク

## 3. Sky Backdrop（js/moge_post.js `fillBackdrop`、モードが Sky のとき）

- 手順2で無効になった画素（空など）を、
  **max(有効画素の最大深度 × 2, 100m) の一定Z平面**としてレイ上に置き直す
- 変更するもの: その画素の depth と points（=幾何のみ）
- 変更しないもの: 色・UV・有効画素
- これで全画素が有効になる。空と前景の境界は以後、普通の「深度の大段差」として扱われる

## 4. EdgeSnap = 深度の中間値の除去（js/edgesnap.js）

- エッジ検出: 隣接画素の深度差が **Edge Threshold**（UI）を超える場所
- 検出したエッジ周辺のランプ画素（ボケた中間深度）を、**元の深度に近い側の台地の
  深度へ吸着**させる: `2 2 3 4 5 5 → 2 2 2 5 5 5`
- 伝播の上限幅が **Snap Width**（UI）。**上限を超えた広いランプは中間深度のまま残る**
  （コンソールの `[EdgeSnap] unresolved` がその画素数。削除はしない）
- 変更するもの: depth と points（レイ上の前後移動のみ）
- **変更しないもの: UV・テクスチャ・画素の有効/無効**

## 5. ColorPatch = テクスチャの混色の塗り直し（js/colorpatch.js）

- 写真のエッジ部分のテクスチャには手前と奥の**混ざった色**（ボケ・アンチエイリアス）が
  写っている。これをエッジ帯だけ**元画像解像度で**塗り直す
- 帯の各画素は「自分と同じ深度側（差10%以内）の帯の外の画素」から色をもらう
  → 手前の帯は手前の色だけ、奥の帯は奥の色だけになり、**色がシームを越えない**
- 同じ側に届かない画素は元の色のまま残す（フォールバックで埋める実験は
  効果がなくロールバック済み。PLAN_EDGE_COLOR.md 3.7）
- 変更するもの: **表示/backfill 用テクスチャ画像の画素の色だけ**
- **変更しないもの: UV・depth・points・メッシュ。元画像そのものも非破壊**
  （Original ダウンロードは原本のまま）

## 6. World Position 化（js/worldpos.js）

- カメラ空間の points を X,Y 反転して Y-up のワールド座標へ（座標系変換のみ）
- 無効画素（mask 除去。Sky モードでは存在しない）は NaN になる

## 7. メッシュ生成 + シーム分割（js/viewer.js `createMesh` / `splitDiscontinuousFaces`）

- world position のグリッドに 1画素=1頂点 のメッシュを張る
- **UV は全頂点「元画像のその画素の位置」のまま**（一度 UV を吸着元へ差し替える方式を
  試したが、モデル解像度粒度のブロック/スジが出たため廃止 → 色の問題は手順5で解決）
- 深度段差が 10%（固定）を超えるセルは**面を削除せず** near / far の2枚に分割:
  - 手前プレートと奥プレートがそれぞれ相手側へ**1セルずつ延長**し合う
    （正面から隙間ゼロ、覗くと切り離されている）
  - この延長用の**複製頂点だけ**は「自分のプレート側の角の UV」をコピーする
    （元の位置の UV のままだと相手側の色を拾ってフリンジになるため。3.6 で確認済み）
- NaN を含むセルだけ面を張らない

## 8. Backfill = 遮蔽穴の裏打ちレイヤー（js/backfill.js）

- シームの**奥側**エッジを種に、深度を奥へ伸長した第2レイヤー（別メッシュ）を作り、
  覗き込んだとき手前プレートの裏に見えるようにする
- 色は手順5の**パッチ済み画像**をモデル解像度に縮小し、種の色をプルプッシュで
  引き延ばす（→ 現状のグレーのスミアはこのプルプッシュの限界。■02 で inpaint 化予定）
- 主メッシュには一切手を加えない

## 早見表: どの段階が何を変更するか

| 段階 | depth / points | UV | テクスチャ画素 | 面（メッシュ） |
|---|---|---|---|---|
| 2. mask 適用 | 無効化のみ | - | - | - |
| 3. Sky Backdrop | 空画素を最奥平面へ | - | - | - |
| 4. EdgeSnap | ランプ画素を台地へ吸着 | **変更しない** | **変更しない** | - |
| 5. ColorPatch | **変更しない** | **変更しない** | エッジ帯のみ塗り直し | - |
| 7. シーム分割 | - | 複製頂点のみプレート側をコピー | - | 段差セルを2枚に分割 |
| 8. Backfill | 第2レイヤーのみ | - | 第2レイヤー用に生成 | 第2レイヤーのみ |

## UI パラメータと対応段階

| UI | 効く段階 |
|---|---|
| Edge Threshold | 4（エッジ検出。1.000=Off で 4,5 を無効化） |
| Snap Width | 4（吸着の伝播上限。超えた分は中間深度のまま= unresolved） |
| Sky / Masked Area | 2, 3 |
| Fill Occlusion / Fill Margin | 8 |

固定しきい値（UIと非連動・変更禁止）: シーム分割 0.10 / backfill 種検出 0.10 /
ColorPatch 帯・側判定 0.10。
