# TASK G: シルエットエッジの前後スパイク（櫛状の膜）解消

2026-07-10 策定。コードレビューと Depth (EXR) の実測から原因を特定し、段階的に対処する。

## 症状

- シルエット境界（視線に対しほぼ平行な深度エッジ）で、メッシュが前後へ櫛状に伸びる
- ワイヤーフレームで見ると、崖の間に引き伸ばされた細長い三角形が行/列ごとに交互に出る
- モーションブラー・ボケの大きい写真ほど悪化する

## 確定した事実（2026-07-10 調査）

- **Depth (EXR) ダウンロードは最終 depth そのもの**（`currentPost.depth` を保存。
  JBU 高解像度化 → mask → Sky Backdrop → EdgeSnap まで適用済み）。メッシュはこれから作られる。
  Initial Depth (EXR) は JBU フィルタ前の初期リサイズ結果（デバッグ用）
- ユーザー提供の EXR 断面（horror-child-stars_01）で、**崖面に 1〜2px 幅の中間深度の歯**が
  シルエットに沿って残っていることを確認。これが頂点になり前後スパイクになる
- [edgesnap.js](../../js/edgesnap.js) は `Snap Width`（既定 8 パス、両側から計 ~16px）を超えた
  ランプ画素を「元の中間深度のまま」残す設計。ボケ幅 20px 超のエッジで中央帯が必ず残る
- しきい値ギャップ: EdgeSnap は Edge Threshold（既定 0.050）で整形するが、
  面分割（viewer.js `splitDiscontinuousFaces`）・backfill 種検出・ColorPatch は **0.10 固定**。
  相対段差 0.05〜0.10 のエッジは「整形されたのに切られない」膜として残る
- 面分割の near/far 2値分類はセル毎に独立（幾何平均しきい値）で、エッジ沿いの整合制約が無い
  → 分類の揺れが櫛状を増幅
- JBU は RGB ガイドなので、テクスチャのボケ幅ぶんの深度ランプは高解像度化で必ず再生産される
- meshopt リダクションはインデックス間引きのみで座標を変えないため原因ではないが、
  スパイク頂点が大きな三角形の角になり見た目を増幅する（TARGET_RATIO 0.05）

## サブタスク

### G-1: EdgeSnap 未解決画素の無効化（実装済み・実機確認待ち）

- `Snap Width` パス終了後に残った pending（flagged だが台地深度が届かなかった画素）を
  **無効化**する（吸着で誤った深度を与えるより、面を張らず backfill に裏打ちさせる。
  三重点セル skip と同じ思想）
- EdgeSnap が更新済み `validMask` を返し、main.js が `cleanedMask` を差し替える
  → WorldPos で NaN → 主メッシュは面を張らない → backfill が holeMask として埋める
- `[EdgeSnap]` ログの `unresolved` に加え `invalidated` を出す
- 切替定数 `INVALIDATE_UNRESOLVED`（edgesnap.js 先頭）。false で従来動作
- 懸念: 樹木の葉など高周波深度領域で flagged クラスタが広いと穴が増える。
  実機で葉・髪の毛の劣化を確認すること

### G-6: ソフトランプの走査検出（実装済み・実機確認待ち）

- 2026-07-10 実機確認: G-1 だけでは**変化なし**。毛・草などソフトな輪郭が背景と重なる場所の
  深度は「数十px のなだらかな S 字ランプ」（1画素あたり 1〜2%）で、隣接対検出（5%）にも
  半径3px の窓検出（両側12%）にも掛からない。flagged にならないので G-1 も発火しない
- 対策: 行/列走査で「同符号の disparity 勾配が連続する run」を追い、
  **長さ `RAMP_MAX_PX`(48) 以下 かつ 総 disparity 変化 > `RAMP_TOTAL_K`(0.25) × 中央値
  disparity** の run だけを flagged に追加する（step 下限 `RAMP_STEP_MIN_K`=0.003×中央値/px）
- **しきい値は視差（parallax）基準**（2026-07-10 ユーザー指示）。切り離すべきかは深度比では
  なく「視点を動かしたときの画面上のズレ量 ∝ disparity 差」で決まる。遠景は深度比2倍でも
  視差ほぼ無し（フラグ不要）、近景は小さな深度比でも膜が目立つ。backfill メッシュの
  視差カットと同じ「シーン中央値 disparity 比」でスケール不変にする
- 本物の斜面（床・壁）との区別は勾配ではなく**長さ**で行う: 膜になるランプは短距離で
  大変化して台地で終わる。床は同じ勾配が何百px も続くので run 長で除外される
- フラグ後は既存機構に乗る: Snap Width まで台地へ吸着 → 残り（幅広ランプの中央帯）は
  G-1 で無効化 → backfill が裏打ち。**Fill Occlusion ON が前提**
- **両端条件（2026-07-10 実機フィードバック反映）**: 近接シーン（ケーキの例）で、布の
  折り目・接地際など**連続面の短い急斜面**を膜と誤判定して穴を開けた。膜と折り目は run の
  「終わり方」で区別する — 膜は両端が台地（勾配ほぼ0）で終わり、折り目・谷・尾根は
  符号反転で終わる（下り→谷底→上り）。**両端とも台地かデータ境界で終わる run だけ**を
  フラグする。副次効果として細い毛・枝（下り+上りが隣接）も保護される
- 懸念1（緩和済み）: 細い毛・枝は両端条件により原則保護される。ただしノイズで run が
  途中分断されると検出漏れが戻る。チューニングは `RAMP_STEP_MIN_K` / `RAMP_MAX_PX` /
  `RAMP_TOTAL_K`
- 懸念2: Sky モードで空が画面の過半を占めると中央値 disparity が空側（極小）へ落ち、
  しきい値が緩みすぎて近景の微妙な起伏まで拾う可能性。空の多い画像で悪化する場合は、
  中央値をパーセンタイル（例: p75）にするか backdrop 画素をサンプルから除外する

### G-2: 主メッシュ面分割の視差基準化（未着手・方針変更）

- 当初案「0.10 固定を Edge Threshold に連動」から変更。**主メッシュの
  `splitDiscontinuousFaces`（相対深度比 0.10 固定）を、backfill メッシュと同じ
  「disparity 差 > k × シーン中央値 disparity」に置き換える**（2026-07-10 ユーザー指示の
  視差基準を分割にも適用）。遠景は深度比が大きくても切らず（視差無し・穴も開かない）、
  近景は小さな深度比でも切る
- near/far 2値分類・複製頂点の延長は現行ロジックを流用し、セルの seam 判定だけ差し替える
- **backfill 種検出（jumpTol 0.10 相対）も同じ視差基準に揃える**。「面が切れた所に種が立つ」
  関係が崩れると未充填の隙間が出るため、必ず同時に変更する
- G-6 の snap 後の崖は、近景では 0.10 相対を下回る場合があり現行分割では繋がったまま残る
  （中央値の 25% ≒ 近景 disp 5×中央値では相対 5%）。視差基準化で解消される
- 影響範囲: viewer.js / main.js（しきい値の受け渡し）/ backfill.js（jumpTol）

### G-3: 引き伸ばし率カット（最終安全網・未着手）

- 面の「深度スパン ÷ スクリーン footprint」（≒ 視線と面のなす角）がしきい値を超える面を
  除去する。1セル=1px なので `fx·|Δz|/z > STRETCH_MAX` 相当の判定で安価に計算できる
- 原因が EdgeSnap 漏れでもアップサンプル起因でも一括で拾える。
  「視野角の高いエッジを伸ばさない」という要件の定義そのものを判定基準にする
- 置き場所: `splitDiscontinuousFaces` のセル判定 or `removeInvalidAndDiscontinuousFaces`

### G-4: near/far 分類のエッジ沿い整合（保留）

- 吸着先選択・2値分類にエッジ接線方向の多数決（1D メディアン）を1パス入れ、櫛を除去
- G-1〜G-3 で十分なら着手しない

### G-5: backfill 奥方向の抑制（チューニング・未着手）

- Far Clamp 既定 4.0x → 2.0x、平面フィット外挿クランプ ±50% → ±30% の検討
- UI スライダーで実機確認してから既定値を変える

## 検証手順（ユーザー実機）

1. スパイクが出る画像でコンソール `[EdgeSnap]` の `unresolved` / `invalidated` を確認
2. ワイヤーフレーム表示で崖の櫛状スパイクが消えているか
3. 葉・髪など高周波領域で穴・欠けが増えていないか（backfill が埋めるか）
4. Depth (EXR) を再ダウンロードし、崖面の中間深度の歯が消えているか
5. 悪化する場合は `INVALIDATE_UNRESOLVED = false` で従来動作へ戻せる
